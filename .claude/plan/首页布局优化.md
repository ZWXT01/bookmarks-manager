**规划时间**: 2026-03-11

## 1. 功能概述

### 1.1 目标

将书签管理器首页从"左侧分类栏 + 右侧内容区"布局重构为"顶部分类导航 + 精简侧边栏 + 主内容区"布局，解决当前分类选择不便的问题，提升用户体验。

### 1.2 范围

**包含**:
- 分类栏从左侧边栏移至主内容区顶部（水平标签页）
- 子分类通过下拉面板展示（点击父分类展开）
- 侧边栏精简为：导入、工具、模板、快捷操作
- 新增全屏分类管理弹窗（支持搜索、拖拽排序、批量操作）
- 响应式适配（移动端下拉菜单 + 抽屉式侧边栏）

**不包含**:
- 后端 API 修改（所有接口保持不变）
- 数据库 schema 变更
- 分类数据结构调整（继续使用 categoryTree）
- 新增业务功能

### 1.3 技术约束

- **技术栈**: EJS 模板 + Alpine.js + Tailwind CSS
- **兼容性**: 保持所有现有功能可用
- **性能**: 不增加额外 HTTP 请求
- **可回滚**: 每个阶段都可以快速回退

## 2. 核心设计方案

### 2.1 新布局结构

```
+----------------------------------------------------------------+
|  Header (固定顶部)                                                |
+------------------------------------------------------------------+
|  +---------------+  +----------------------------------------+ |
|  |  Sidebar      |  |  Main Content                            | |
|  |  (精简版)     |  |                                          | |
|  |               |  |  +------------------------------------+  | |
|  | 📥 导入       |  |  | 分类导航栏 (水平滚动)                | |
|  | 🛠️ 工具       |  |  | [📚全部] [📂未分类] [📁分类1] ...   |  | |
|  | 🗂️ 模板       |  |  +------------------------------------+  | |
|  |               |  |                                          | |
|  | 快捷操作:     |  |  +------------------------------------+  | |
|  | + 添加分类    |  |  | ➕ 添加书签                          | |
|  | + 批量检查    |  |  +------------------------------------+  | |
|  |               |  |                                          | |
|  +---------------+  |  +------------------------------------+  | |
|                     |  | 🔍 搜索书签                          |  | |
|                     |  +------------------------------------+  | |
|                     |                                          | |
|                     |  +------------------------------------+  | |
|                     |  | 书签列表 (可滚动)                    |  | |
|                     |  +------------------------------------+  | |
+------------------------------------------------------------------+
```

### 2.2 分类导航设计

**一级分类**：水平标签页，无需滚动即可看到所有分类
**子分类**：点击父分类时在下方展开面板
**分类管理**：独立的全屏弹窗，支持搜索、批量操作

## 5. 风险控制

### 5.1 潜在风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 分类过多导致水平滚动不便 | 中 | 添加左右滚动按钮、支持鼠标滚轮横向滚动 |
| 子分类面板遮挡内容 | 低 | 使用半透明遮罩、点击外部自动关闭 |
| 移动端操作不便 | 中 | 增大点击区域、使用原生下拉菜单 |
| 旧版用户不适应新布局 | 低 | 添加"布局切换"功能、首次使用显示引导 |

### 5.2 回滚方案

每个 Phase 都可以独立回滚：
- Phase 1: 删除新增的 CSS 类和数据字段
- Phase 2: 隐藏新组件，显示旧组件
- Phase 3: 恢复旧布局结构
- Phase 4: 恢复被删除的代码
