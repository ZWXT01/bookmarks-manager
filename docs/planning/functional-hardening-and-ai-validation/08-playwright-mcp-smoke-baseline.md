## 4. 选择器策略

- 优先使用可访问语义选择器：
  - `getByRole(...)`
  - `getByPlaceholder(...)`
  - `getByText(...)`
- 页面级断言优先顺序：
  - 路径
  - 页面标题
  - 稳定文案关键字
- 对设置页、AI 配置页、Token 页等可能带敏感值的页面：
  - 不在日志或文档中输出输入框当前值。
  - 不依赖完整 DOM dump 做断言。
  - 优先只记录路径、标题、按钮文案和区块标题。
- 不使用：
  - 仓库内 Playwright spec 文件。
  - 易漂移的 Tailwind class 选择器。
  - 依赖具体数据条数或具体书签标题的断言。

## 5. 留痕规则

- 成功路径默认只记录：
  - 访问路径
  - 页面标题
  - 关键断言是否通过
  - 控制台错误 / 警告摘要
- 失败路径才保留额外证据：
  - 1 张截图或 1 份 accessibility snapshot
  - 对应控制台错误摘要
  - 失败步骤、目标路径和失败断言
- 若页面含有密码、API key、Token 或其他敏感值：
  - 不保留展示敏感值的截图。
  - 不把页面完整 DOM 输出到文档。
  - 必要时只保留脱敏后的错误摘要。

## 6. 清理规则

- 结束验证后关闭 MCP 浏览器会话。
- 若 `/tmp/playwright-mcp-output/` 下生成了本次会话的截图、console log 或其他产物，任务结束前删除本次会话目录。
- 本 issue 不启动项目内 Playwright，不残留 `npx playwright` 进程。
- 本 issue 使用人工提供的在线测试环境，不启动本地临时服务，因此不产生额外端口占用。

## 8. 发现的问题

- 首页对 `OverlayScrollbars` 的 CSS / JS 使用了错误的 `integrity` 值，浏览器在测试环境中阻断了资源加载。
- 站点缺少 `favicon.ico`，会产生 `404` 控制台错误。
- `tailwind.js` 仍以运行时 CDN 方式使用，浏览器控制台会持续提示生产环境警告。

这些问题不阻断当前最小 smoke，但会影响“零控制台错误”的发布质量，已转入风险台账继续跟踪。
